4 Migrate Data

Migrate all current data from local JSON files to the database. This includes:

Users & Config
  • User accounts (username, display name, admin status)
  • BitBucket credentials
  • Per-user configuration settings
Projects & Assignments
  • Master project list (name, folder, notes)
  • User–project assignments with status
  • Activity log history
Deploy the DB structure first. Existing records will be updated (not duplicated) using ON DUPLICATE KEY UPDATE.
1 Data Storage Backend

Choose where all application data is stored and read from. When Database is active, all reads and writes (users, projects, assignments, activity log) go to the configured MariaDB database.

Local (JSON Files)

Data stored in /opt/pulpitans/ JSON files. Default behaviour, no DB required.

Database

All data stored in the configured MariaDB database. Requires DB connection to be set up.

Switching to Database mode requires the DB to be configured and the structure deployed. Go to Database tab.
What is stored in the database
Users
  • Username, display name
  • Admin status
  • BitBucket credentials
  • Per-user config
Projects
  • Project name, folder
  • Notes
Assignments
  • User → Project
  • Status (done, wip, etc.)
Activity Log
  • All user actions
  • Timestamps
